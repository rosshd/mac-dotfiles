================================================================================
HOW THE PIECES FIT TOGETHER (read this first)
================================================================================

When you open your Mac, this is the chain of tools at play:

  KARABINER remaps Caps Lock into a "super key" that no other app will
  accidentally steal, making all your shortcuts one physical key.

  AEROSPACE is your window manager — it arranges every app window into a
  tiling grid automatically and lets you switch between workspaces and move
  windows using Caps+hjkl.

  ITERM2 opens and immediately hands control to TMUX, which lives inside it.
  You're never really using "iTerm" directly — you're using tmux sessions
  that happen to render inside iTerm.

  TMUX gives you persistent sessions (they survive closing iTerm), split
  panes, and multiple windows inside one terminal. All pane movement uses
  Ctrl+hjkl so your hands never leave the home row.

  FISH is your shell — the thing that actually runs your commands. Starship
  draws the prompt. Aliases shorten common commands. On every new tmux window
  the custom FASTFETCH animation plays to show system info.

  VS CODE is your editor for all actual coding. It opens files, runs a
  terminal panel inside it, and uses GitLens + Pylance + ESLint to understand
  your code. AI tools (Claude Code, Codex) run inside its terminal panel.

  RAYCAST replaces Spotlight — it's faster and smarter for launching apps,
  math, clipboard history, and anything you'd normally Cmd+Space for.

  GIT / GH CLI handle all version control from the terminal. Starship shows
  git status in your prompt at all times so you never forget what branch
  you're on.

Day-to-day flow:
  Open iTerm → tmux auto-attaches → fastfetch plays → you're at a prompt.
  Caps+e opens VS Code. Caps+1/2/3/4/5 switch workspaces.
  Ctrl+\ or Ctrl+g splits a terminal pane. Ctrl+hjkl moves between panes.
  Ctrl+o drops a Claude Code session into a split. Caps+i does inline AI in VS Code.

SHELL — Fish
================================================================================

WHAT IT IS:
  Fish (Friendly Interactive Shell) is your command line shell — the program
  that reads what you type and runs it. It replaces bash/zsh as your default.

WHY FISH OVER BASH/ZSH:
  Fish has autosuggestions built in (grey text shows the most likely completion
  as you type — press → to accept). Syntax highlighting works as you type so
  typos turn red before you run them. No config required for basic use.

WHERE IT SHOWS UP:
  Everywhere you type a command. Every terminal pane in tmux runs fish.
  All aliases and startup logic live here.

Config: ~/.config/fish/config.fish

What's configured:
  - PATH: ~/.local/bin (wrappers) → /opt/homebrew/bin → pyenv
  - pyenv initialized so `python3` uses the right version
  - SSH agent auto-started so `git push` never asks for a passphrase
  - Starship prompt initialized
  - Fish greeting suppressed (no "Welcome to fish" message)
  - Fastfetch runs automatically on new tmux windows (not splits)

Aliases — what they do and when you use them:
  dev         jump to ~/Developer/projects    (where your repos live)
  school      jump to ~/School                (coursework)
  sandbox     jump to ~/Developer/sandbox     (throwaway experiments)
  g           git                             (saves 2 chars every command)
  gs          git status                      (most typed git command)
  ga          git add                         (stage files)
  gc          git commit                      (commit staged files)
  gp          git push                        (push to remote)
  gl          git log ...                     (visual branch graph, last 15)
  gd          git diff                        (see unstaged changes)
  ll          ls -lah                         (list files with sizes + hidden)
  ..          cd ..                           (go up one folder)
  ...         cd ../..                        (go up two folders)
  grep        grep --color=auto               (grep with color always on)
  py          python3                         (shorter)
  vi          nvim                            (opens neovim in terminal)
  matrix      cmatrix -a -C cyan              (matrix rain screensaver)
  pipes       pipes.sh -t 0                   (animated pipes screensaver)
  spaceship   ~/.config/fish/spaceship.py     (full-screen spaceship animation)
  fastfetch   launch.py --no-timeout          (show system info, stay open)

PROMPT — Starship
================================================================================

WHAT IT IS:
  Starship is the program that draws your command prompt — the line you type
  commands after. It replaces the default boring "user@host $" with something
  that shows actually useful context.

WHY IT MATTERS:
  At a glance you can see what folder you're in, what git branch you're on,
  whether there are uncommitted changes, how long the last command took, and
  what time it is. You never need to type `git status` just to check.

WHERE IT SHOWS UP:
  Every time a command finishes and fish is waiting for your next input.

Config: ~/.config/starship.toml
Theme: orange/space, two-line layout

What you see:
  Line 1: ╭ ~/path/to/folder   branch ~2 +1   took 3s   14:22
  Line 2: ╰❯ (where you type)

What each piece means:
  ~/path/to/folder    current directory (truncated to 4 levels deep)
   branch            git branch name (purple)
  ~2                  2 files modified (not yet staged)
  +1                  1 file staged (ready to commit)
  ⇡3                  3 commits ahead of remote (need to push)
  took 3s             last command took 3 seconds (only shows if > 2s)
  14:22               current time

Language versions appear automatically when relevant:
   3.11.2            shows in Python projects (or when venv is active)
   1.21.0            shows in Go projects
   20.0.0            shows in Node.js projects

================================================================================

WHAT IT IS:
  iTerm2 is your terminal emulator — the macOS app that provides the window
  and rendering for your shell. Think of it as the frame around everything.

WHY ITERM2 OVER TERMINAL.APP:
  Better font rendering, transparency/blur, split panes (though you use tmux
  for splits), dynamic profiles configurable as JSON, and iTerm shell
  integration for clickable file paths.

WHERE IT SHOWS UP:
  It's the app you open. After that, you're interacting with tmux inside it.
  iTerm itself is mostly invisible — you set it up once and forget it.

Profile: "Ross" (Dynamic Profile — auto-loaded from Ross.json, no manual setup)
On open: auto-runs `tmux new-session -A -s main` so you always land in tmux.

Visual settings:

  Background:   dark space blue, 15% transparent so wallpaper shows through
  Blur:         10 (subtle depth effect behind the terminal)
  Cursor:       orange blinking block
  Theme:        Minimal (hides the tab bar chrome — must be set manually in
                Preferences → Appearance → Theme → Minimal)

Useful native iTerm features (rarely needed since tmux handles most of this):

  Hold Cmd      click a file path or URL to open it

================================================================================
TERMINAL MULTIPLEXER — tmux
================================================================================

WHAT IT IS:
  tmux (Terminal MUltipleXer) runs inside iTerm and manages multiple terminal
  sessions. It's like a window manager specifically for your terminal.

WHY IT MATTERS:
  Sessions persist even if iTerm closes or your SSH connection drops. You can
  split one terminal window into many panes (e.g., code on the left, server
  on the right, AI assistant on the far right). Named sessions let you keep
  separate workspaces (e.g., one session per project).

WHERE IT SHOWS UP:
  You're always inside tmux when using iTerm. The orange borders around panes
  and the status bar at the bottom are tmux. Every split, every window tab,
  every "the terminal didn't die when I closed it" moment is tmux.

Config: ~/.tmux.conf
Prefix: Ctrl+a (rarely needed — most things work without it)

NO-PREFIX KEYBINDS (just press these, no Ctrl+a first):
  Ctrl+\        split current pane → new pane to the RIGHT
  Ctrl+g        split current pane → new pane BELOW
  Ctrl+t        new tmux window (like a new tab)
  Ctrl+n        next tmux window
  Ctrl+p        previous tmux window
  Ctrl+x        close current pane (asks confirmation)
  Ctrl+h        move focus LEFT to adjacent pane
  Ctrl+j        move focus DOWN to adjacent pane
  Ctrl+k        move focus UP to adjacent pane
  Ctrl+l        move focus RIGHT to adjacent pane
  Ctrl+o        open Claude Code in a new split pane
  Ctrl+y        open Codex in a new split pane

WITH PREFIX (Ctrl+a, release, then press):
  H/J/K/L       resize current pane (hold to repeat)
  S             list all sessions and switch between them
  X             kill the current window
  r             reload tmux config without restarting
  |             split right (backup to Ctrl+\)
  -             split down (backup to Ctrl+g)

Visual:
  Status bar:    bottom strip — shows session name, window list, path, time
  Active pane:   bright orange border + "● foldername" at top
  Inactive pane: amber border + "○ foldername" at top
  Window names:  auto-set to the current folder name

Fastfetch:
  Plays automatically on every new tmux window (Ctrl+t or first open).
  Does NOT replay when splitting panes.
  Exits cleanly on resize, leaving a clean prompt.

WINDOW MANAGER — AeroSpace
================================================================================

WHAT IT IS:
  AeroSpace is an i3-inspired tiling window manager for macOS. It automatically
  arranges app windows into a grid instead of letting them float wherever.

WHY IT MATTERS:
  Without a window manager, you're constantly dragging and resizing windows.
  With AeroSpace, every app snaps into a tiling layout. You switch focus and
  move windows without touching the mouse. Workspaces let you group apps
  (e.g., workspace 1 = code, workspace 2 = browser, workspace 3 = terminal).

WHERE IT SHOWS UP:
  Running invisibly at all times. Every time you use a Caps+key shortcut to
  focus a window, move it, or switch workspace, that's AeroSpace.

Config: ~/.config/aerospace/aerospace.toml
Layout: tiles, 8px gaps between windows

ALL KEYBINDS (Caps = Caps Lock held, remapped by Karabiner):

  App launch:
    Caps+Enter        open iTerm
    Caps+e            open VS Code
    Caps+b            open Firefox

  Focus (move cursor between windows):
    Caps+h            focus window to the LEFT
    Caps+j            focus window BELOW
    Caps+k            focus window ABOVE
    Caps+l            focus window to the RIGHT

  Move (drag window to new position):
    Caps+Shift+h      move window LEFT
    Caps+Shift+j      move window DOWN
    Caps+Shift+k      move window UP
    Caps+Shift+l      move window RIGHT

  Workspaces:
    Caps+1 to Caps+5  switch to workspace 1–5
    Caps+Shift+1..5   move focused window to that workspace
    Caps+Tab          jump back to the last workspace you were on

  Layout controls:
    Caps+/            rotate tile direction (horizontal ↔ vertical)
    Caps+Space        toggle between floating and tiling for current window
    Caps+f            fullscreen current window
    Caps+q            close focused window
    Caps+=            make current window larger
    Caps+-            make current window smaller
    Caps+r            reload AeroSpace config file

KEY REMAPPING — Karabiner-Elements
================================================================================

WHAT IT IS:
  Karabiner-Elements is a low-level keyboard remapper for macOS. It intercepts
  keystrokes before any app sees them and can transform them into something else.

WHY IT'S HERE:
  AeroSpace needs a modifier key combination that nothing else will accidentally
  trigger. Ctrl+Option+Cmd (three modifiers at once) is essentially unused by
  every other app. Karabiner makes Caps Lock send that combination when held,
  so physically you just press Caps+something and it works.

WHERE IT SHOWS UP:
  Invisibly, always running. Every Caps+key shortcut you press goes through
  Karabiner first, which converts it to Ctrl+Option+Cmd+key for AeroSpace.

Config: ~/.config/karabiner/karabiner.json
Profile: "Ross"

The single rule:
  Caps Lock HELD   → Ctrl+Option+Cmd  (AeroSpace trigger key)
  Caps Lock TAPPED → Escape           (useful in vim mode, modal editors)

Nothing else is remapped. Option, Ctrl, Cmd all work normally.

SETUP NOTE: After installing, open Karabiner-Elements once and approve
  accessibility permission in System Settings → Privacy & Security → Accessibility.
  Without this approval the remap silently does nothing.

PYTHON — pyenv
================================================================================

WHAT IT IS:
  pyenv manages multiple Python versions on one machine. Different projects
  can require different Python versions — pyenv handles switching automatically.

WHY IT MATTERS:
  macOS ships with a system Python you should never modify. pyenv gives you
  isolated, controllable Python installs that don't affect the system.

WHERE IT SHOWS UP:
  Whenever you run `python3` or `pip`. The version in use shows in your
  Starship prompt when you're inside a Python project.

Useful commands:
  pyenv install --list          see all available versions
  pyenv install 3.12.0          install a specific version
  pyenv global 3.12.0           use this version everywhere by default
  pyenv local 3.11.0            pin this directory to a specific version
                                (creates .python-version file, auto-applied)
  pyenv versions                list all installed versions
  pyenv version                 show currently active version and why

Virtual environments (use these to isolate project dependencies):
  python3 -m venv .venv                  create a venv in current folder
  source .venv/bin/activate.fish         activate it in fish
  pip install [package]                  install into the venv only
  deactivate                             leave the venv

================================================================================
THINGS TO LEARN (once workflow is settled)
================================================================================

These are all installed and configured. Learn them in roughly this order:

  1. AeroSpace workspaces
       Practice using Caps+1..5 to separate concerns:
       workspace 1 = code, 2 = browser, 3 = terminal, etc.
       Learn Caps+Shift+1..5 to move windows between them.

  2. tmux sessions
       Beyond splits and windows: named sessions per project.
       `tmux new -s projectname`, detach with Ctrl+a d, reattach with `tmux a -t projectname`.
       Lets you context-switch between projects without closing anything.

  3. fzf (fuzzy finder)
       In fish: Ctrl+R for fuzzy history search, Ctrl+T for fuzzy file search.
       Pipe anything into it: `git branch | fzf`, `ls | fzf`, etc.

  4. ripgrep (rg) + fd
       `rg "function name"` — searches all files instantly, respects .gitignore.
       `fd filename` — finds files instantly, cleaner than `find`.
       These replace grep and find for all practical purposes.

  5. GitLens in VS Code
       Hover any line to see who wrote it and when.
       Cmd+Shift+P → "GitLens: Show File History" for full history of any file.
       "Open Changes" to see a visual diff of any commit.

  6. fish functions (beyond aliases)
       `abbr` expands text inline as you type (e.g., type "gs" and it expands
       to "git status" in the command line before running).
       Functions can take arguments: `function mkcd; mkdir $argv; cd $argv; end`

  7. pyenv local + venv workflow
       Pin each project to its Python version with `pyenv local 3.x.x`.
       Always create a .venv in project root. The Starship prompt shows you
       when a venv is active. Never install packages globally.

  8. gh CLI deeper features
       `gh pr create --fill` — auto-fills PR title/body from commit messages.
       `gh run list` — see CI status. `gh run watch` — stream CI logs live.
       `gh repo fork` — fork a repo and clone it in one command.
